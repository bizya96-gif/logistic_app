# Logistics App

Дашборд учета рейсов для Google Таблицы `2026 Загрузки`.

## Состав проекта

- `Проект логистики.html` - HTML-разметка приложения.
- `styles.css` - стили приложения поверх Bootstrap.
- `app.js` - клиентская логика, интеграция с Apps Script, фильтры, таблица и форма рейса.
- `apps-script/Code.gs` - код Google Apps Script Web App для чтения и записи в таблицу.
- `Протокол проекта в логистике 10.06.docx` - описание архитектуры, бизнес-логики и плана развития.
- `Проект логистики.backup-before-safe-changes.html` - резервная копия HTML до безопасных правок от 24.06.2026.

## Важные ограничения

- Не менять структуру колонок Google Таблицы без синхронной правки Apps Script и фронтенда.
- Не менять бизнес-формулы ID, расходов и прибыли без отдельной проверки на реальных данных.
- POST-запросы сейчас работают через `mode: 'no-cors'`, поэтому фронтенд не читает ответ записи. Успех подтверждается повторным GET и проверкой данных.
- Статус оплаты чекбоксом записывается через Apps Script в колонку P (`Дата оплаты`). При чтении рейс считается оплаченным, если P заполнена или сумма в колонке M имеет черный цвет текста.
- Оранжевый акцент является текущим визуальным стандартом проекта.
- Apps Script использует `SpreadsheetApp.getActiveSpreadsheet()`, поэтому должен быть привязан к рабочей Google Таблице.
- Финальный ID нового рейса генерируется в Apps Script под `LockService`; ID в форме является предварительным.

## Apps Script

Серверный код находится в `apps-script/Code.gs`. При изменениях его нужно вручную перенести в редактор Google Apps Script или настроить отдельную синхронизацию.

Поддерживаемые действия:

- `GET` - чтение рейсов из листа `2026 Загрузки`.
- `POST action=create` - добавление рейса.
- `POST action=update` - обновление рейса по ID.
- `POST action=delete` - удаление рейса по ID.
- `POST action=updatePayment` - запись или очистка даты оплаты в колонке P.
