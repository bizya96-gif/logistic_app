# Logistics App

Дашборд учёта рейсов на основе Google Таблицы `2026 Загрузки`. Сайт не заменяет таблицу — оба инструмента используются параллельно, часть пользователей продолжает работать напрямую в Google Таблице.

**Статус:** актуализировано 09.07.2026. Этот файл — единственный источник правды по проекту (протокол `project-protocol.docx` от 10.06 устарел и оставлен только как архивная историческая запись, см. его содержимое).

## Состав проекта

- `index.html` - разметка приложения.
- `styles.css` - стили поверх Bootstrap.
- `app.js` - клиентская логика: интеграция с Apps Script, фильтры, таблица, форма рейса.
- `apps-script/Code.gs` - код Google Apps Script Web App для чтения и записи в таблицу.
- `project-protocol.docx` - архивный документ (устарел, не использовать как источник правды).

В таблице есть строки без ID (запланированные рейсы) и служебные строки с формулами-нулями внизу — их нельзя удалять, они нужны для сводки.

## Бизнес-логика

- **ID рейса:** `[Первая буква авто]-[порядковый номер]`, напр. `Г-106`. Генерируется в Apps Script под `LockService`; ID в форме — предварительный, финальный присваивается на сервере. При редактировании ID не меняется.
- **Прибыль:** `Прибыль = Цена договора − (На топливо + Платная дорога + Комиссионные)`. Считается автоматически; при ручном вводе авто-расчёт фиксируется (отключается), при очистке поля — возвращается.
- **Оплата:** статус хранится в самой таблице (не в localStorage). Чекбокс пишет дату оплаты в колонку P через `action=updatePayment`; колонка M (сумма) при этом красится в чёрный (оплачено) или красный (не оплачено). При чтении рейс считается оплаченным, если P заполнена ИЛИ цвет шрифта M — чёрный.
- **Вставка нового рейса:** Apps Script находит последнюю строку с реальными данными и вставляет новый рейс сразу после неё, не затрагивая формулы внизу таблицы.
- **Не менять структуру колонок Google Таблицы** без синхронной правки Apps Script и фронтенда.
- **Не менять формулы ID/расходов/прибыли** без проверки на реальных данных.
- POST-запросы идут через `mode: 'no-cors'` — фронтенд не читает ответ записи напрямую, успех подтверждается повторным GET.

## Apps Script (`apps-script/Code.gs`)

Поддерживаемые действия:

- `GET` - чтение рейсов; кэшируется на 60 секунд (`CacheService`), сброс кэша — параметром `?noCache=1`. Необязательный параметр `?sheet=НазваниеЛиста` позволяет разово прочитать архивный лист (напр. `2025 Загрузки`, `2024 Загрузки`) для выгрузки в статичный JSON — такие запросы не пишутся в общий кэш и не влияют на чтение текущего года.
- `POST action=create` - добавление рейса.
- `POST action=update` - обновление рейса по ID (запись по фиксированным номерам колонок).
- `POST action=delete` - удаление рейса по ID.
- `POST action=updatePayment` - запись/очистка даты оплаты в колонке P + перекраска суммы в M.
- `POST action=patchTrip` - точечное обновление конкретных полей рейса по названию колонки (используется отдельно от `update`, дублирует часть его функциональности).

Apps Script использует `SpreadsheetApp.getActiveSpreadsheet()` — скрипт должен быть привязан к рабочей Google Таблице.

**Синхронизация кода:** при изменениях в `Code.gs` его нужно вручную перенести в редактор Google Apps Script (или настроить `clasp`) — автоматической синхронизации между репозиторием и задеплоенным скриптом нет. Если правите скрипт прямо в Google, обязательно переносите изменения обратно в этот файл, иначе репозиторий разойдётся с продакшеном (уже происходило).

## Известные ограничения (актуально на 09.07.2026)

1. **Авторизации на уровне Apps Script нет** — эндпоинт открыт для любого, кто знает URL. Осознанное решение при текущем масштабе (1-5 пользователей, публичная ссылка не раздаётся); стоит пересмотреть при росте числа пользователей или публикации ссылки.
2. **Кэш не инвалидируется при ручном редактировании таблицы** — если кто-то правит строку прямо в Google Sheets (не через сайт), дашборд может показывать устаревшие данные до 60 секунд.
3. **Два механизма записи** (`update` по номерам колонок и `patchTrip` по названиям) дублируют функциональность — риск, что один сломается при изменении структуры таблицы, а другой нет.
4. **Сайт не имеет постоянного хостинга** — сейчас запускается только локально; выбранный целевой вариант — GitHub Pages.
